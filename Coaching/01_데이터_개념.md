# 제1강 데이터

## 1. 데이터의 개념

### 데이터의 의미

- 데이터(Data)는 '주어진 것'을 의미하는 라틴어 'Datum'에서 유래했습니다.
  - Data는 Datum의 복수형입니다.
 
- 데이터는 '현실 세계에서 발생한 현상을 숫자, 문자, 소리, 이미지, 영상 등의 형태로 기록한 가공되지 않은 자료'라는 의미로 널리 사용됩니다.
  - 무엇이든지간에 그것이 기록되었다면 데이터라고 할 수 있습니다. 세상에는 거의 무한한 데이터가 존재합니다.

- 지식의 계층 구조를 설명하는 DIKW 피라미드 이론에 따르면 데이터는 피라미드의 가장 아래 부분에 위치합니다.
  - 데이터(Data)를 가공하면 정보가 되고, 정보(Infomation)를 분석해서 체계화하면 지식이 되고, 지식(Knowledge)을 유연하게 적용하여 판단을 내리면 지혜(Wisdom)가 됩니다.

### 쓸모있는 데이터의 특징

- 데이터라고 해서 모두 다 쓸모가 있는 것은 아닙니다.
  - 그래서 데이터의 개념을 아는 것만큼이나 어떤 데이터가 쓸모있는 데이터인지 파악하는 것이 중요합니다.

- 우리의 삶을 긍정적인 방향으로 바꾸는 데 도움을 줄 수 있는 데이터를 쓸모있는 데이터라고 할 수 있습니다.
  - 데이터가 우리의 삶에 긍정적인 영향을 미칠 수 있으려면 데이터는 어떤 특성을 가지고 있어야 할까요?

- 제가 생각한 결과는 다음과 같습니다.
  - 첫째, 데이터가 현실을 왜곡하지 않아야 합니다.
  - 둘째, 의사결정에 활용할 수 있어야 합니다.
  - 셋째, 사용하기에 너무 어렵지 않아야 합니다.

## 2. 데이터 기반 의사결정

### 데이터 기반 의사결정의 필요성

- 우리는 살아가면서 수많은 의사결정을 합니다.
  - 의사결정에 따라 성공과 실패가 겪게되고, 그 성공과 실패는 우리의 삶에 영향을 줍니다.

- 최대한 우리가 원하는 결과를 얻기 위해 노력하지만, 결과를 미리 알 수는 없습니다.
  - 데이터 기반 의사결정은 이러한 불확실성을 줄여주는 강력한 도구입니다.
  - 의사결정은 반복되기 때문에 과거의 데이터를 분석해서 미래의 의사결정을 합리적으로 바꿔나갈 수 있습니다.

### 데이터 기반 의사결정을 위해 필요한 것

- 데이터 기반 의사결정을 하려면 데이터와 판단 기준이 필요합니다.
  - 첫째, 의사결정을 위해 참고할 데이터가 있어야 합니다. 의사결정을 위해 참고하려면 데이터가 단순히 존재하기만 해서는 부족하고, 표나 그래프, 보고서 등의 형태로 가공되어 있어야 합니다. 
  - 둘째, 의사결정의 결과를 판단할 기준이 필요합니다. 반복되는 의사결정을 합리적으로 개선하기 위해서는 과거의 의사결정이 성공적이었는지 아닌지 판단해야 합니다. 그것은 단순히 '성공'/'실패'로 나누어 질 수도 있고, 점수 형태로 기록될 수도 있습니다. 필요하다면 문장이나 그림, 영상 형태로 기록될 수도 있지만, 아직까지 문장, 그림, 영상 자체를 기준으로 삼는 알고리즘은 개발되지 않았기 때문에 성공/실패로 나누거나 몇 개의 지표를 기준으로 판단할 수 있어야 합니다.

## 3. 고용노동부의 데이터

### 데이터의 양과 질

- 고용노동부는 여러 부처들 중에서도 상당히 많은 양의 데이터를 가지고 있는 부처입니다.
  - 고용노동부가 제공하는 정책과 서비스는 국민 전체를 대상으로 하는 경우가 많고, 정책 집행 과정에서 수집, 보관, 이용하는 정보도 매우 다양합니다.
  - 노동법 위반, 산업재해 발생 및 보상, 노동조합의 설립과 운영, 비정규직의 근로조건, 외국인 노동자 현황, 임금·퇴직연금 및 근로시간, 고용보험과 산재보험, 노동시장 통계조사 등 고용노동부만 다룰 수 있는 정보도 많습니다.
  - 근로복지공단, 산업안전보건공단, 산업인력공단과 같은 산하기관도 규모가 크고, 추진하는 사업도 많습니다.

- 다만, 데이터의 양에 비해 질 관리 수준은 미흡한 점이 많습니다.
  - 첫째, 데이터가 목적에 따라 분산되어 있어 통합적이고 체계적인 데이터 관리가 이뤄지지 못하고 있습니다.
  - 둘째, 메타데이터 관리가 부실하기 때문에 여러 데이터를 연계하려고 해도 상당한 시간과 노력이 필요하고, 때로는 데이터 연계가 불가능합니다.
  - 셋째, 데이터 관리 인력과 조직이 턱없이 부족합니다. 고용노동부 전체 인력규모에 비해 데이터 관리 인력은 채 1%가 되지 않습니다.

### 일하는 방식의 문제

- 고용노동부의 일하는 방식도 개선할 필요가 있습니다.
  - 현장 중심의 조직문화가 오랜 기간 정착되어 왔기 때문에 데이터의 중요성을 제대로 인식하고 있지 못합니다.
  - 당장 현장에서 문제가 없다면 데이터 관리가 다소 부실하더라도 문제삼지 않으며, 국회나 언론에 대응하기 위해 내부 통계시스템을 사용하지 않고, 그때그때 별도의 공문이나 메일을 보내 현황을 파악하는 것을 당연하게 생각합니다.

- 데이터를 생산하는 방식도 엑셀 중심으로 이뤄지고 있어 시스템 전환의 걸림돌이 될 때가 많습니다.
  - 중요한 데이터를 엑셀로만 관리하게되면 데이터의 이력관리나 신뢰성 확보에 한계가 있을 수 밖에 없는데, 이러한 문제를 실무담당자의 책임으로 돌리는 관행이 다수 존재합니다.
