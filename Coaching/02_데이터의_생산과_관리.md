# 제2강 데이터의 생산과 저장

## 1. 데이터의 생산

### 데이터를 생산하는 이유

- 우리는 업무를 하면서 다양한 데이터를 생산합니다.
  - 데이터를 생산하는 이유는 다양하지만, 가장 중요한 이유는 기록을 남기기 위해서 입니다.
  - 우리가 어떠한 업무를 수행했어도 기록이 남아있지 않다면 그 일을 하지 않은 것과 같습니다.

- 우리가 업무를 하면서 생산한 데이터들은 분석의 대상이 되기도 합니다.
  - 우리는 스스로 생산한 데이터를 분석하고, 그 안에서 의미를 찾아냅니다.
  - 데이터가 의미하는 바를 찾아서 더 나은 방식으로 업무를 수행하는 것을 데이터 기반 행정이라고 합니다.

- 우리는 업무의 기록을 남기기 위해서, 그리고 그 기록을 바탕으로 더 나은 방식으로 업무를 하기 위해서 데이터를 생산합니다.

### 데이터를 생산하는 방식

- 데이터를 생산하는 방식에는 크게 3가지 방법이 있습니다.

- 첫 번째는 사람이 직접 생산하는 것입니다.
  - 손으로 적기, 키보드나 마우스로 입력하기, 음성 녹음, 영상 촬영 등 다양한 방법을 사용할 수 있습니다.
  - 어떤 방식을 쓰더라도 사람이 직접 데이터를 생산한다면 첫 번째 방식입니다.

- 두 번째 방법은 전산 시스템을 이용하는 것입니다.
  - 사람이 모든 데이터를 일일히 입력하게 하지않고, 전산 시스템이 데이터를 생산하도록 합니다.
    - 예를 들어, 도서관에서 책을 빌릴 때 책의 바코드를 찍기만 하면 시스템이 알아서 대여한 책 제목, 대여시간, 반납여부 등의 데이터를 입력해 줍니다.
  - 이 방법은 첫 번째 방법과 함께 사용되는 경우가 많습니다.

- 세 번째 방법은 센서를 이용한 방법입니다.
  - 센서는 주변 환경의 변화를 파악할 수 있는 전자기기입니다. 기계나 장치에 설치된 센서들은 매순간 데이터를 생산할 수 있습니다.
    - 요즘 기계들에는 대부분 수십 개 이상의 센서가 포함되어 있으며, 이 센서들은 기계가 작동할 때마다 수 많은 데이터를 생산합니다.
  - IoT(Internet of Things)라는 개념은 모든 제품들이 센서를 가지고 상호 작용할 수 있다는 전제하에 만들어진 개념입니다.

- 각각의 데이터 생산 방법은 각각 장단점이 있으며, 이를 적절하게 활용해야 효율적으로 데이터를 생산할 수 있습니다.

- 데이터를 생산하는 방법 중 가장 피해야 하는 방법은 모든 데이터를 사람이 직접 생산하도록 만드는 것입니다.
  - 다르게 표현하면, 고용노동부 직원이나 민원인에게 데이터를 입력해야 하는 의무를 부과하는 방식으로 제도를 설계해서는 안됩니다.
  - 예를 들어, 실업급여 담당자가 상담을 제대로 하는지 파악하기 위해서 의무적으로 상담일지를 작성하도록 하거나, 민원인에게 구직활동 내역을 작성하도록 하는 것은 비효율적입니다.
  - 사람들의 행동이 자연스럽게 데이터 생산으로 이어지도록 제도와 시스템을 설계해야 합니다. 이를 위해서는 웹사이트나 앱을 통해 사람들이 원하는 서비스를 받을 수 있도록 하고, 그 과정에서 자연스럽게 데이터가 생산될 수 있도록 해야 합니다.

### 데이터를 생산할 때 지켜야 하는 원칙

- 데이터를 아무렇게나 생산하는 것은 잘못된 행동입니다. 생각보다 많은 사람들이 이 사실을 모릅니다.
  - 잘못 생산된 데이터는 다른 데이터까지 신뢰할 수 없게 만듭니다.
  - 마치 잘 정리된 방에 누군가 들어와서 아무렇게나 물건을 쌓아놓으면 정리되어 있던 물건들까지 찾기 어려워지는 것과 같습니다.

- 데이터를 생산할 때는 다음과 같은 원칙을 지켜야합니다.
  - 첫째, 데이터 표준을 지켜야 합니다.
  - 둘째, 비논리적인 데이터를 생산하면 안됩니다.
  - 셋째, 입력된 데이터를 삭제하지 않아야 합니다.

- 데이터 생산 원칙을 지키지 않고 생산된 데이터는 활용하기가 매우 어렵습니다.
  - 원칙을 지키지 않고 생산된 데이터를 지저분한 데이터라고 표현하며, 잘못 생산된 데이터를 원칙에 맞게 수정하는 작업을 데이터 클리닝이라고 합니다.

### ① 데이터 표준의 준수

- 데이터 표준을 지키는 것은 데이터 생산의 첫번째 원칙입니다.
  - 데이터 표준은 모든 데이터에 공통적으로 적용되어야 합니다.
  - 데이터 표준은 각 기관별로, 시스템별로, 업무별로 다를 수 있습니다. 표준이 여러 개라면 모든 표준을 준수해야 합니다.

#### 데이터를 생산해야 하는데, 데이터 표준이 없다면?

- 데이터 표준이 아예 존재하지 않는다면, 곤란한 상황입니다.
  - 데이터 표준이 없을 때 여러분이 할 수 있는 선택은 크게 3가지 입니다.
- 최악의 선택은 표준이 없으니까 '아무렇게나' 데이터를 생산하는 것입니다.
  - 처음에 일을 할 때는 괜찮을 수 있지만, 데이터가 쌓이면 쌓일수록 그 데이터는 지저분한 데이터가 됩니다.
  - 지저분한 데이터는 활용하기가 어렵고, 만드는 과정이 아무리 힘들었다고 하더라도 데이터로서의 가치가 매우 낮습니다.
  - 안타깝게도 우리부의 데이터 중 상당수가 이런 상태입니다. 표준에 대한 고민없이 당장의 문제를 해결하기 위해 데이터를 생산하면 이런 결과가 초래됩니다.
- 최선의 선택은 데이터 표준을 만드는 것입니다.
  - 업무의 성격이 다르더라도 데이터 표준은 크게 다르지 않습니다. 다 비슷비슷합니다. 특별한 양식도 없습니다. 데이터를 생산할 때 지켜야 하는 내용들을 기록해두면 그것으로 충분합니다.
  - 다른 데이터 표준을 참고하여 새롭게 데이터 표준을 만드는 것이 어려운 작업은 아닙니다.(하루 이틀 정도면 ok)
  - 꽤 많은 경우에는 존재하는 다른 업무의 데이터 표준을 그대로 사용하는 것도 가능합니다.
  - 필요하다면 데이터 업무를 수행하는 부서(지능정보화기획팀, 고용서비스기반과 등)의 도움을 받을 수도 있습니다.
- 현실적인 선택은 대부분의 경우에 공통적으로 적용되는 데이터 표준의 원칙을 고려하여 데이터를 생산하는 것입니다.
  - 데이터 표준을 만들어서 적용하지 않았더라도, 기본적인 원칙만 지킨다면 시간이 지나 누군가 데이터를 정비하려고 할 때 들여야하는 시간과 노력을 크게 줄일 수 있습니다.
  - 데이터를 전산시스템에 저장한다면 기본적인 원칙을 대부분을 지킬 수 밖에 없지만, 시스템을 구축할 때까지 기다릴 수 없을만큼 상황이 급해서 엑셀로 데이터 관리를 하고 있다면 이러한 원칙들이 무시될 수 있습니다.

#### 데이터 생산의 원칙(엑셀파일 관리법)

- 만약 여러분이 업무 때문에 엑셀파일로 데이터를 관리하기 시작했다면 다음 원칙들을 지켜야 합니다.

##### 파일 관리

- 첫째, 파일은 날짜별로 생성합니다.
  - 데이터를 엑셀 파일로 저장하고 있다면, 파일명은 날짜로 시작하고 작성자로 끝나야 합니다.
  - "260228_데이터_역량강화교육_참여자(김O현).xlsx" 과 같이 파일명을 정해야 합니다.
  - 그리고 나중에 해당 파일을 수정했다면 수정한 날짜 기준으로 파일을 다시 생성합니다. 파일이 수십개, 아니 수백수천개가 생기더라도 괜찮습니다. 
    날짜별로 파일을 만드세요.
  - 하루에도 몇 번씩 파일을 수정한다면 _v1, _v2 을 붙여서 구분하셔도 좋습니다.

- 둘째, 생성된 파일은 별도의 장소에 보관합니다.
  - 생성된 파일이 내 컴퓨터에만 저장되어 있으면 절대 안 됩니다.
  - 메일을 보내든, G-Drive에 올리든 해서 꼭 별도의 백업이 되어 있어야 합니다.

##### 데이터 입력 양식

- 첫째, 데이터는 행 방향으로 기록해야 합니다.
  - 엑셀 파일은 행과 열로 이루어진 2차원 행렬 구조 입니다.
  - 데이터 입력할 때 지켜야할 가장 기본적인 원칙은 행 방향으로(위에서 아래로) 데이터를 입력해야 한다는 것입니다.
  - 교육과정에 참여한 사람을 이름과 소속을 기록하는 엑셀파일이라면 이런 형태여야 합니다.

|소속|이름|
|:-:|:-:|
|혁신행정담당관|김OO|
|혁신행정담당관|이OO|
|기획재정담당관|이OO|
|지능정보화기획팀|박OO|
|기획재정담당관|송OO|
|지능정보화기획팀|최OO|

  - 열 단위로 입력하는 방법은 다른 게 아니라 **잘못된** 방법입니다.

- 둘째, 데이터는 A1부터 입력합니다.
  - 데이터에는 반드시 제목과 내용이 있습니다.
    - 제목은 제일 앞에 있다고 해서 헤더(header), 내용은 밸류(value) 라고 부르기도 합니다.
    - 위의 표에서는 '이름'과 '소속'이 헤더입니다.
  - 헤더는 1열에 적습니다.
    - 가끔씩 보기 좋게 하려고 A열이나 1행을 비워두고 B2부터 데이터를 입력하는 경우가 있는데, **하지 마세요**. 엑셀 그렇게 쓰는거 아닙니다.
    - 헤더를 2열, 3열까지 늘리는 것도 안 하는 것이 좋습니다.
      - 헤더에 내용이 많더라도 1열에 다 적는게 훨씬 좋습니다.

- 셋째, 셀병합을 하지 않습니다.
  - 엑셀 파일은 행과 열로 이루어진 2차원 행렬 구조 이고, 모든 기능은 그 전제 하에서 작동합니다.
    - 셀 병합을 하는 것은 어떤 이유에서든지 **정말 아주 매우** 잘못된 행동입니다.
  - 셀 병합을 해야 할 것 같다는 생각이 들면 병합 셀에 들어갈 내용을 반복해서 작성해야 합니다.
    - 셀 병합 하지 말고 다음과 같이 입력하십시오.

|국|과|이름|
|:-:|:-:|:-:|
|정책기획관|혁신행정담당관|김OO|
|정책기획관|혁신행정담당관|이OO|
|정책기획관|기획재정담당관|이OO|
|정책기획관|지능정보화기획팀|박OO|
|정책기획관|기획재정담당관|송OO|
|정책기획관|지능정보화기획팀|최OO|

- 넷째, 하나의 셀에는 하나의 값만 입력합니다.
